# 📊 Data Preprocessing

## Overview

Data Preprocessing is the **first and most important step** in any Machine Learning pipeline. 
It involves cleaning, transforming, and organizing raw data into a suitable format for model training.

This module focuses on essential preprocessing techniques used in real-world ML workflows.

---

## 📁 Folder Structure

This section contains structured implementations of key preprocessing techniques:

- Handling Missing Values
- Encoding Categorical Data
- Feature Scaling
- Outlier Detection
- Feature Engineering
- Train Test Split


Each topic includes:

- 📘 theory.md → Concept explanation  
- 📓 .ipynb → Interactive experiments & visualization  
- 🐍 .py → Reusable Python implementation  

---

## 📌 Topics Covered

### 1. Handling Missing Values
Techniques to deal with incomplete data such as:
- Mean / Median / Mode imputation
- Forward fill / Backward fill
- Interpolation
- Deletion methods

---

### 2. Encoding Categorical Data
Converting categorical variables into numerical form using:
- Label Encoding
- One-Hot Encoding
- Ordinal Encoding

---

### 3. Feature Scaling
Normalizing numerical data for better model performance:
- Standardization (Z-score)
- Min-Max Scaling
- Robust Scaling

---

### 4. Outlier Detection
Identifying and handling extreme values using:
- IQR method
- Z-score method
- Boxplots

---

### 5. Feature Engineering
Creating meaningful features from raw data:
- Feature transformation
- Feature creation
- Feature selection basics

---

### 6. Train-Test Split
Splitting dataset into training and testing sets:
- Random splitting
- Stratified splitting
- Cross-validation basics

Data preprocessing is crucial because:

- Real-world data is messy  
- Models cannot handle missing or categorical data directly  
- Proper preprocessing improves accuracy and performance  
- It is the foundation of all ML pipelines
